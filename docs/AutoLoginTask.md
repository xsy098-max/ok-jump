# AutoLoginTask 使用说明

`AutoLoginTask` 是 `ok-jump` 项目中用于处理游戏启动与自动登录流程的核心任务模块。它能够识别多种登录界面状态，支持自动输入账号，并处理常见的登录交互。

## 📋 功能特性

*   **自动启动游戏**: 可配置是否由脚本启动游戏客户端。
*   **多场景识别**: 支持识别“适龄提示页”、“账号登录页”、“服务器选择页”等多种登录阶段界面。
*   **智能交互**:
    *   优先使用图像特征匹配查找“进入游戏”/“开始游戏”按钮。
    *   特征匹配失败时，自动回退到 OCR 文字识别进行点击。
*   **账号自动输入**:
    *   支持自动点击输入框并输入预设账号。
    *   内置输入校验机制（通过剪贴板比对），确保输入内容准确无误。
    *   支持输入失败重试。
*   **异常处理**:
    *   自动检测“登录失败”、“网络错误”等弹窗。
    *   遇到错误时自动保存截图并记录日志。

## ⚙️ 配置说明

配置文件路径: `configs/AutoLoginTask.json`

| 配置项 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `启用` | boolean | `true` | 是否启用自动登录任务。 |
| `自动启动游戏` | boolean | `false` | 是否由脚本自动启动游戏程序（需在全局配置中设置游戏路径）。 |
| `等待游戏启动(秒)` | integer | `120` | 等待游戏窗口出现的最长超时时间。 |
| `最大登录尝试次数` | integer | `5` | 尝试点击登录按钮的最大次数。 |
| `输入账号` | boolean | `false` | 是否执行账号输入流程。 |
| `账号` | string | `""` | 需要输入的账号文本。 |
| `输入后缓冲时间(秒)` | float | `0.8` | 账号输入完成后的等待缓冲时间。 |
| `账号输入重试次数` | integer | `2` | 账号输入校验失败时的重试次数。 |
| `输入校验超时(秒)` | float | `1.0` | 校验输入框内容是否正确的超时时间。 |
| `登录等待超时(秒)` | integer | `60` | 整个登录流程的最长等待时间。 |
| `点击后等待时间(秒)` | integer | `3` | 点击登录按钮后的等待页面响应时间。 |

### 示例配置

```json
{
    "启用": true,
    "自动启动游戏": true,
    "等待游戏启动(秒)": 120,
    "最大登录尝试次数": 6,
    "输入账号": true,
    "账号": "my_account_name",
    "输入后缓冲时间(秒)": 1.0,
    "账号输入重试次数": 3,
    "输入校验超时(秒)": 2.0,
    "登录等待超时(秒)": 60,
    "点击后等待时间(秒)": 5
}
```

## 🖥️ 运行流程

1.  **任务启动**: 检查任务是否启用，如已登录则直接跳过。
2.  **启动游戏**: 若配置了 `自动启动游戏`，尝试启动游戏进程。
3.  **等待窗口**: 轮询检测游戏窗口是否存在。
4.  **场景识别与交互**:
    *   **场景 0 (适龄提示)**: 识别并点击“进入游戏”。
    *   **场景 1 (账号登录)**:
        *   若需输入账号且未完成，识别输入框并执行输入。
        *   点击“进入游戏”。
    *   **场景 3 (服务器选择)**: 点击“开始游戏”。
    *   **未知场景**: 尝试通过 OCR 寻找“进入游戏”、“开始游戏”或“登录”字样进行点击。
5.  **结果判定**:
    *   成功进入大厅或游戏内 -> **登录成功**。
    *   检测到错误提示或超时 -> **登录失败**（保存截图）。

## ⚠️ 注意事项

*   **账号输入框定位**: 账号输入依赖于 `assets/images/login/input.png` 模板图片来定位输入框。如果游戏更新导致UI变化，可能需要更新此图片。
*   **输入法**: 自动输入时建议保持系统输入法为英文状态，以免输入干扰。
*   **OCR依赖**: 该任务严重依赖 OCR 识别按钮文字，请确保 OCR 环境配置正确。
