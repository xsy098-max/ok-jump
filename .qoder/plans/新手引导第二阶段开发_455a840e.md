
# 自动新手引导第二阶段开发计划

## 完整流程概览

```
Phase1结束
  ↓
点击"开始对战"(end02.png)
  ↓
加载界面 #1 (OCR百分比检测, 停滞60s抛错)
  ↓ [15秒容错]
加载界面 #2 (OCR百分比检测, 重置百分比追踪)
  ↓
检测"积分争夺"(fight_start.png)
  ↓
启动自动战斗(AutoCombatTask) → 持续检测"对战结束"(fight_end), 最多3分30秒
  ↓ [20秒过渡容错]
MVP场景: 检测"点击荧幕退出"(out.png) → 点击屏幕中心
  ↓
加载界面 #2.5 (OCR百分比检测)
  ↓
再次检测"点击荧幕退出"(out.png) → 再次点击屏幕中心
  ↓ [15秒过渡容错]
新英雄场景: 检测"新英雄"(new_hero.png) + "确定"(comfirm.png) → 点击确定
  ↓ [短暂容错]
加载界面 #3 (OCR百分比检测)
  ↓ [10秒容错]
主界面: 检测"漫斗赛" + "排位赛"(最多重试3次)
  ↓
第二阶段结束 → 停止所有任务, GUI保持打开
```

## Task 2: 创建 Phase2Handler（核心实现）

### 文件：`src/tutorial/phase2_handler.py`（新建）

参照 `phase1_handler.py` 的架构风格，实现以下内部子步骤：

#### 2.1 点击"开始对战"
- 主检测：模板匹配 `end02.png`（Features.TUTORIAL_END02）
- 备选检测：OCR 识别"开始对战"文字（简繁双语：开始对战 / 開始對戰）
- 检测到后点击该按钮位置
- 容错：循环检测直到按钮消失，确认点击成功

#### 2.2 双加载界面等待
- **复用** `TutorialDetector` 中已有的 `detect_loading_start()` / `detect_loading_end()` / `_detect_loading_percentage()` 方法
- 第一个加载：等待百分比出现 → 监控变化 → 等待结束（含60秒停滞检测）
- 两个加载之间：15秒容错窗口，超时报错并截图
- 第二个加载：**重置百分比追踪状态**后重新检测（因为第二个加载百分比会比第一个结束时的小）
- 同样继承停滞超时抛错机制

#### 2.3 战斗开始检测
- 主检测：模板匹配 `fight_start.png`（Features.TUTORIAL_FIGHT_START）
- 备选检测：OCR 识别"积分争夺"文字（简繁双语：积分争夺 / 積分爭奪）
- 任一方式命中即视为检测成功
- 检测到后标记战斗开始状态

#### 2.4 启动自动战斗 + 并行结束检测
- 参照 `phase1_handler.py` 中 `_handle_combat_trigger()` 的方式，直接创建 `AutoCombatTask` 实例
- 继承 AutoCombatTask.json 中所有战斗设置（普攻、技能、间隔等）
- 启动独立后台线程运行自动战斗
- **并行检测线程**：持续检测"对战结束"
  - 主检测：模板匹配 `fight_end.png`（Features.TUTORIAL_FIGHT_END）
  - 备选检测：OCR 识别"对战结束"文字（简繁双语：对战结束 / 對戰結束）
  - 任一方式命中即视为检测成功
- 最大等待 3 分 30 秒（210 秒），超时报错并截图

#### 2.5 战斗结束处理
- 检测到"对战结束"后：
  - 停止自动战斗线程（设置退出标志）
  - 停止移动控制器、技能控制器等战斗相关操作
  - 标记战斗结束状态
  - 日志记录战斗持续时间

#### 2.6 MVP场景处理（包含两次点击）

**第一次 MVP 点击：**
- 20 秒容错过渡等待（战斗结束 → MVP 出现）
- 主检测：模板匹配 `out.png`（Features.TUTORIAL_MVP_OUT）
- 备选检测：OCR 识别"点击荧幕退出"文字（简繁双语：点击荧幕退出 / 點擊熒幕退出）
- 任一方式命中即视为检测成功
- 超时 20 秒报错并截图
- 检测到后点击屏幕中心坐标

**中间加载界面：**
- 点击后进入一次加载界面，复用 OCR 百分比检测 + 停滞超时机制
- 重置百分比追踪状态

**第二次 MVP 点击：**
- 加载结束后，再次检测"点击荧幕退出"（同样模板 + OCR 双模式）
- 检测到后再次点击屏幕中心坐标
- 超时 20 秒报错并截图

#### 2.7 新英雄场景处理
- 15 秒容错过渡等待（MVP 点击后 → 新英雄界面）
- 在 20 秒内同时检测两个元素（两者都需命中）：
  - 元素1：模板匹配 `new_hero.png`（Features.TUTORIAL_NEW_HERO） / 备选 OCR"新英雄"（简繁双语：新英雄 / 新英雄）
  - 元素2：模板匹配 `comfirm.png`（Features.TUTORIAL_CONFIRM_BUTTON） / 备选 OCR"确定"（简繁双语：确定 / 確定）
- 两个元素都检测到后，点击"确定"按钮位置
- 超时报错并截图

#### 2.8 最终加载界面
- 短暂容错后检测加载界面开始
- 复用相同的 OCR 百分比检测 + 停滞超时机制
- 加载结束后 10 秒容错缓冲

#### 2.9 主界面验证
- OCR 检测"漫斗赛" + "排位赛"（简繁双语）
- **两个都检测到**才算成功
- 失败后最多重试 3 次
- 全部重试失败则报错截图

#### 2.10 日志规范
- 所有日志带 `[第二阶段]` 前缀
- 支持详细日志模式（verbose）
- 错误时自动截图保存（带时间戳命名）

## 风险和注意事项

1. **自动战斗线程管理**：Phase 2 的自动战斗需确保能干净退出，避免线程泄漏
2. **双加载界面的百分比重置**：第二个加载的百分比比第一个结束时小，必须重置追踪状态
3. **简繁适配**：所有 OCR 文字检测必须经过 LangConverter 处理，支持简繁双语
4. **全部角色模式**：本次不开发，作为后续独立迭代任务
