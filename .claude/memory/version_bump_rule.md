## 规则

每次完成功能更新或 bug 修复后，自动递增 `config.py` 中的 `version` 字段：
- **小改动**（bug 修复、小功能调整）→ 递增修订号（patch）：`1.4.30` → `1.4.31`
- **大改动**（新功能模块、重大重构）→ 递增次版本号（minor）：`1.4.30` → `1.5.0`

版本号仅在 `config.py` 第 74 行的 `'version'` 字段定义，无需修改其他文件。

**Why:** 用户希望版本号能反映每次代码变更，且项目版本号与 GUI 工具显示的版本号同步（它们共用同一个 config）。

**How to apply:** 在每次功能修改完成后，自动修改 `config.py` 中的 `version` 值。不要询问用户是否需要更新版本号，直接执行。判断标准：补丁/bug修复 → patch+1；新任务/新模块/重大变更 → minor+1。
