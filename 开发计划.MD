# 《漫画明星：大集结》自动化测试工具开发计划 (ok-jump)

## 1. 项目架构设计文档

### 1.1 项目概述
本项目旨在基于 `ok-script` 框架，构建针对 MOBA 手游《漫画明星：大集结》（Jump: Assemble）的自动化测试与辅助工具。项目采用分层架构设计，确保模块的高内聚低耦合，并充分利用 `ok-script` 提供的图像识别、OCR 和任务调度能力。

### 1.2 核心模块设计

#### 1.2.1 游戏状态识别模块 (Game State Recognition)
- **核心类**: `JumpScene` (继承自 `ok.BaseScene`)
- **功能**: 实时检测游戏当前所处的界面（主界面、选人界面、加载中、战斗中、结算界面）。
- **实现方式**:
  - **特征匹配**: 使用 `find_feature` 识别关键UI元素（如“开始游戏”按钮、战斗HUD图标）。
  - **OCR文本识别**: 识别特定场景标题或状态文本。
  - **状态机**: 维护 `current_scene` 状态，处理场景流转逻辑。

#### 1.2.2 操作模拟模块 (Action Simulation)
- **核心类**: `BaseJumpTask` (继承自 `ok.BaseTask`)
- **功能**: 封装游戏内的基础交互逻辑。
- **实现方式**:
  - **输入封装**: `tap()`, `swipe()`, `send_key()` (映射键盘到触屏点击)。
  - **导航逻辑**: `navigate_to(scene_name)`，实现从任意界面跳转到目标界面的路径规划。
  - **异常处理**: 处理点击无效、网络延迟导致的弹窗干扰。

#### 1.2.3 战斗系统模块 (Combat System)
- **核心类**: `CombatTask`, `HeroBase`
- **功能**: 实现自动战斗逻辑。
- **实现方式**:
  - **英雄抽象**: `src/char/` 下定义不同英雄的技能连招策略。
  - **目标检测**: 集成 YOLO (参考 `ok-wuthering-waves`) 或 OpenCV 模板匹配识别敌方血条/小兵。
  - **技能循环**: 根据 CD 监控（像素点颜色变化或 OCR）执行技能释放。

#### 1.2.4 测试用例与任务管理 (Test Case Management)
- **配置管理**: `configs/tasks.json` 定义任务参数。
- **任务调度**: 利用 `ok-script` 的 `TaskExecutor` 进行任务队列管理。
- **用例实现**: 每个测试场景（如“排位赛自动挂机”、“新手引导测试”）实现为一个独立的 `Task` 类。

#### 1.2.5 结果报告与日志 (Result Reporting)
- **日志系统**: 使用 `ok.og.logger` 记录详细的操作流水和调试信息。
- **截图留证**: 关键节点（如报错、战斗结束）自动触发 `save_screenshot`。
- **报告生成**: 运行结束后生成 Markdown/HTML 格式的测试摘要（胜率、运行时长、异常次数）。

### 1.3 与 ok-script 框架集成
- **入口点**: `main.py` 初始化 `ok.OK(config)`，加载自定义配置。
- **配置注入**: `config.py` 中定义 `ConfigOption`，注入全局设置（如模拟器分辨率、使用的英雄）。
- **目录结构遵循**:
  ```text
  ok-jump/
  ├── assets/             # 图像模板、OCR模型、YOLO模型
  ├── configs/            # 用户配置文件
  ├── src/
  │   ├── char/           # 英雄逻辑
  │   ├── combat/         # 战斗核心算法
  │   ├── scene/          # 场景定义 (JumpScene.py)
  │   ├── task/           # 具体任务实现 (BaseJumpTask.py 等)
  │   └── globals.py      # 全局变量
  ├── tests/              # 单元测试
  ├── main.py             # 启动入口
  ├── config.py           # 配置定义
  └── requirements.txt    # 依赖管理
  ```

## 2. 分阶段功能开发清单

### 第一阶段：基础框架搭建与环境联调 (预计耗时: 3天)
- [ ] **F01-项目初始化**: 建立目录结构，配置 Git 仓库，编写 `.gitignore`。
- [ ] **F02-环境连接**: 调通 `adbutils` 连接模拟器，实现 `ok-script` 基础截图与点击功能。
- [ ] **F03-基础场景识别**: 实现 `JumpScene`，支持识别“登录页”和“主大厅”。
- [ ] **F04-配置系统**: 建立 `config.py`，支持读取模拟器路径和分辨率设置。
- **验收标准**: 运行 `main.py` 能成功连接模拟器，并准确输出当前场景名称日志。

### 第二阶段：核心交互与导航系统 (预计耗时: 5天)
- [ ] **F05-通用任务基类**: 实现 `BaseJumpTask`，封装 `wait_for_feature`, `click_feature` 等通用方法。
- [ ] **F06-场景导航**: 实现主界面到各个子系统（如背包、商城、匹配大厅）的自动跳转。
- [ ] **F07-异常弹窗处理**: 全局监控通用弹窗（签到、公告、网络重连）并自动关闭。
- [ ] **F08-OCR集成**: 集成 `onnxocr`，识别货币数量、活动文本。
- **验收标准**: 编写脚本能从登录界面自动一路点击进入训练场，过程无卡顿，遇弹窗能恢复。

### 第三阶段：战斗系统原型 (预计耗时: 7天)
- [ ] **F09-战斗场景检测**: 识别战斗UI（摇杆、技能按钮、血条）。
- [ ] **F10-基础普攻与移动**: 实现虚拟摇杆控制移动，模拟普攻点击。
- [ ] **F11-技能释放框架**: 建立 `HeroBase` 类，实现技能CD检测与释放逻辑。
- [ ] **F12-简单战斗AI**: 实现“索敌->移动->攻击”的基础循环（基于简单的像素/模板匹配）。
- **验收标准**: 在训练场中，角色能自动寻找人偶并释放一套完整连招。

### 第四阶段：自动化测试套件与完善 (预计耗时: 5天)
- [ ] **F13-每日任务脚本**: 实现“每日签到”、“领取邮件”等简单回归测试用例。
- [ ] **F14-自动匹配战斗**: 实现完整的匹配流程：开始匹配 -> 确认 -> 选人 -> 战斗 -> 结算 -> 返回。
- [ ] **F15-统计报告**: 记录战斗时长、胜负结果，生成运行报告。
- [ ] **F16-GUI配置界面**: 利用 `ok-script` 的 PySide6 支持，暴露常用配置项到界面。
- **验收标准**: 能够连续运行“自动匹配战斗”3次以上不报错，并生成测试日志。

## 4. 开发环境与依赖管理说明

- **Python 版本**: 3.10 或 3.11 (推荐 3.10 以获得最佳兼容性)

## 5. 测试策略与质量保证要求

### 5.1 单元测试 (Unit Testing)
- **覆盖率目标**: 核心工具类（如坐标转换、配置读取、数据解析）代码覆盖率需 **≥80%**。
- **工具**: 使用 `pytest` 运行测试。
- **位置**: `tests/` 目录下对应源码结构编写测试文件（如 `src/utils.py` -> `tests/test_utils.py`）。

### 5.2 集成测试 (Integration Testing)
- **关键路径**: 针对“场景识别”和“配置加载”编写集成测试。
- **Mock**: 使用 Mock 对象模拟 `ok.OK` 上下文，避免测试时必须启动真实游戏客户端。

### 5.3 自动化验收 (Pre-commit)
- **本地检查**: 每次提交代码前，必须在本地运行以下命令并通过：
  ```powershell
  pytest tests/           # 运行单元测试
  python main.py --check  # (可选) 运行自检脚本，检查资源文件完整性
  ```
- **资源检查**: 确保所有新添加的图片模板 (`assets/`) 均已在 `config.py` 或对应的 Feature 定义中注册。

## 6. 交付物清单

项目最终交付（或归档）时需包含以下内容：

1.  **源码 (Source Code)**
    - 完整的 `src/` 目录，包含所有 Python 源代码。
    - 完整的 `assets/` 目录，包含所有用于匹配的图片模板和模型文件。
    - `configs/` 目录下的默认配置文件。

2.  **文档 (Documentation)**
    - `README.md`: 项目介绍、快速开始指南。
    - `开发计划.MD`: 本文档，包含路线图和架构设计。
    - `CHANGELOG.md`: 版本迭代历史。
    - `docs/API.md`: 核心类和方法的接口说明（可选）。

3.  **测试脚本 (Test Scripts)**
    - `tests/` 目录下的所有单元测试代码。
    - `examples/` 目录下提供的示例用法（如 `simple_farm_demo.py`）。

4.  **使用手册 (User Manual)**
    - 简易的用户操作指南（Markdown 或 PDF），说明如何配置模拟器、如何选择英雄、如何启动脚本。

5.  **构建脚本 (Build Scripts)**
    - `requirements.txt`: 依赖清单。
    - `run.bat` / `run.sh`: 一键启动脚本。
